Example Block Library

The topic-dependent half of the system. Where the chrome blocks barely vary page to page, these four vary with what the worked example actually is — a causal chain wants nodes and arrows, a chronology wants a spine, a chart wants an axis. Each still renders inside the chrome's .figure-box, which already handles the border, background, and caption; these components only fill the content area.

Class-naming convention: .ex-<pattern>. Four are built here — .ex-mechanism, .ex-compare, .ex-timeline, .ex-datadive — covering mechanism walkthrough, comparative pair, timeline of one event, and data deep-dive. The menu's other twelve formats can follow the same convention when they need one (.ex-artifact, .ex-caselaw, …) — but not all of them will. A documented exchange or an artifact close-reading is often just well-set prose; don't force a diagram where the prose already carries it.

Every color below is one of the seven tokens already defined in textbook-chrome-blocks.html — no second palette, no new hex values. .ex-datadive's bars use color-mix() to step between --rose-dust and --rose-deep, so magnitude reads as depth of the same hue rather than a rainbow of unrelated colors.

.ex-mechanism — mechanism walkthrough: a causal chain, traced step by step

Nodes flex-wrap, so a chain of any length reflows instead of overflowing; on narrow screens the arrows rotate 90° to read top-to-bottom. .node-accent (rose) marks the outcome the reader should land on. .node-alt (sage) is a second variant for chains with two distinct compartments or phases — pair it with an optional .phase-label divider (a full-width caption that breaks the row) when the mechanism genuinely has two acts, not just three-plus steps in one.

.ex-compare — comparative pair: two real things, one isolated variable

Deliberately not the chrome .compare table — that's for a generic multi-attribute spec sheet. This is a pair: two cards that mirror each other's fact list line for line, with the one thing that actually differs named out loud in the center badge, so the reader isn't left to guess which row is the point. Stacks to a single column under 560px, badge falling between the two cards in reading order.

.ex-timeline — timeline: one real event's chronology

A fixed-width date column keeps every date right-aligned against the spine regardless of length ("1648" and "14 July 1789" both sit flush). .dot-accent marks the beat that mattered most — the turning point, not necessarily the last one chronologically.

.ex-datadive — data deep-dive: one real dataset, unpacked live

One axis, one series, one hue — bars step from --rose-dust to --rose-deep in proportion to their share of the series max, so color and height encode the same thing instead of fighting. Values are labeled directly on every bar rather than left to a legend, which only works because there are few enough categories to read at a glance; a denser dataset should thin the labels before it thins the bars. Each <rect> carries a native SVG <title> for a free hover tooltip, and the <svg> itself carries a full-sentence aria-label so the trend survives without sight or a mouse. Swap the four bars and the viewBox width for a line-chart <polyline> when the data is closer to continuous (a time series) than categorical — same tokens, same axis rules.
