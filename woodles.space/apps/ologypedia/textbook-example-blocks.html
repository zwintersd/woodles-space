Example Block Library

The topic-dependent half of the system. Where the chrome blocks barely vary page to page, these four vary with what the worked example actually is — a causal chain wants nodes and arrows, a chronology wants a spine, a chart wants an axis. Each still renders inside the chrome's .figure-box, which already handles the border, background, and caption; these components only fill the content area.

Class-naming convention: .ex-<pattern>. Ten are built here, covering all sixteen of the menu's formats — .ex-mechanism, .ex-compare, .ex-timeline, .ex-datadive, .ex-case, .ex-document, .ex-exchange, .ex-spectrum, .ex-envelope, .ex-failure. One pattern can honestly cover more than one menu item when the shapes converge — .ex-case is a single "specific real instance, and what it settled" card reused across legal case, single documented case, policy/institutional episode, and biographical vignette, distinguished only by its eyebrow label. The last three menu items — artifact close-reading, geographic case, experiment walkthrough — are demonstrated at the bottom of this file too, but deliberately without a new component: one reaches for .ex-case again, one composes three existing blocks with no wrapper of its own, and one is a two-line typographic convention that isn't part of the .ex- family at all. Don't build a pattern where reuse already carries it.

Every color below is one of the seven tokens already defined in textbook-chrome-blocks.html — no second palette, no new hex values. .ex-datadive's bars and .ex-spectrum's track use color-mix() / a two-token gradient rather than any new hue; .ex-envelope borrows var(--font-mono) from shared/fonts.css (already loaded) for tabular digit alignment — still no font stack of its own.

.ex-mechanism — mechanism walkthrough: a causal chain, traced step by step

Nodes flex-wrap, so a chain of any length reflows instead of overflowing; on narrow screens the arrows rotate 90° to read top-to-bottom. .node-accent (rose) marks the outcome the reader should land on. .node-alt (sage) is a second variant for chains with two distinct compartments or phases — pair it with an optional .phase-label divider (a full-width caption that breaks the row) when the mechanism genuinely has two acts, not just three-plus steps in one.

.ex-compare — comparative pair: two real things, one isolated variable

Deliberately not the chrome .compare table — that's for a generic multi-attribute spec sheet. This is a pair: two cards that mirror each other's fact list line for line, with the one thing that actually differs named out loud in the center badge, so the reader isn't left to guess which row is the point. Stacks to a single column under 560px, badge falling between the two cards in reading order.

.ex-timeline — timeline: one real event's chronology

A fixed-width date column keeps every date right-aligned against the spine regardless of length ("1648" and "14 July 1789" both sit flush). .dot-accent marks the beat that mattered most — the turning point, not necessarily the last one chronologically.

.ex-datadive — data deep-dive: one real dataset, unpacked live

One axis, one series, one hue — bars step from --rose-dust to --rose-deep in proportion to their share of the series max, so color and height encode the same thing instead of fighting. Values are labeled directly on every bar rather than left to a legend, which only works because there are few enough categories to read at a glance; a denser dataset should thin the labels before it thins the bars. Each <rect> carries a native SVG <title> for a free hover tooltip, and the <svg> itself carries a full-sentence aria-label so the trend survives without sight or a mouse. Swap the four bars and the viewBox width for a line-chart <polyline> when the data is closer to continuous (a time series) than categorical — same tokens, same axis rules.

.ex-case — case file: one shape, four menu items (legal case · single documented case · policy/institutional episode · biographical vignette)

Retarget the same three-part shape — header, facts, outcome — by swapping .case-kind's text and the outcome label: Documented Case / "What Happened," Policy Episode / "Downstream Effect," Biographical Vignette / "Why It Matters." All four menu items are "a specific real instance, and what it settled" — they don't need four components, just four labels on one.

.ex-document — primary document excerpt: a real letter, memo, ruling, or transcript

The gold left-rule and italic display font mark this as quoted material, distinct from the narrator's own voice everywhere else on the page — the one place a textbook page should deliberately sound like someone other than the narrator.

.ex-exchange — documented exchange: a real back-and-forth between two people, paraphrased

Left/right alignment does the work a "he said, she said" tag would otherwise carry in prose. Keep turns short and paraphrased — this is a shape for the back-and-forth of an exchange, not a transcript dump.

.ex-spectrum — edge case / boundary stress-test: a real instance that almost doesn't fit

The gradient track is itself the argument: the concept isn't binary, and the edge case's whole interest is in how close it sits to the far pole without quite crossing it. Gold marks a prototypical case for reference; rose-deep marks the edge case under discussion.

.ex-envelope — back-of-envelope math: real numbers, worked live

Every row is one operation and its running value — a reader should be able to redo the math themselves from what's on screen, which is the entire point of a back-of-envelope worked example. Numbers set in var(--font-mono) for tabular alignment; everything else on the page stays in Lora.

.ex-failure — failure case: something real that didn't work, and a documented account of why

Sage marks the path nobody got to see (the intention); rose marks what was actually documented happening. Keeping both visible at once, rather than narrating the failure in isolation, is what makes the gap legible as a gap.

.artifact-meta — artifact close-reading: one real object and what it reveals (not an .ex- pattern on purpose)

Two lines of markup, not a component — a <dl> and a paragraph. If a specific physical feature of the object is doing real argumentative work ("note the wear pattern here, and here"), that's .ex-mechanism borrowed wholesale, with its step numbers read as callouts to features instead of stages in a process — still not a fifth thing to maintain.

.ex-case again — geographic case: a real place where the concept plays out visibly

Zero new CSS on this one — it's literally .ex-case a fifth time, with .case-kind reading "Geographic Case" instead of "Legal Case." A place that did something specific and checkable is structurally identical to a ruling that did: header, facts, outcome.

composed from three .ex- blocks — experiment walkthrough: one real study's design and result

No wrapper class holds these three together — .composed-demo above is nothing but a margin reset between three ordinary block instances, stacked in the order a reader needs them: what was predicted, how it was set up, what happened. A fourth "experiment" pattern would just be these three blocks pre-glued, which is exactly the kind of premature component this library is trying to avoid.

Sixteen menu items, ten shapes — the rest just borrow. ✦
