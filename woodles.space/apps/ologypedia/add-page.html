One Entry at a Time, Start to Finished File

✦

There's no database behind this site, no "save" button writing to a server — Ologypedia is a static block library, on purpose. So a whole textbook gets built the honest way: one entry at a time. Name a topic and this studio hands you the exact prompt to grow it. Paste back what the model writes, and it renders live right here — then downloads as a finished file, named and ready to drop in. Four steps, no backend, and you never leave this page until the entry is done.

1

Describe the Entry

One entry at a time. Name the topic; add a line about your own relationship to it if there is one; flag it if it's a diagnosis or health condition. Everything else — the voice, the structure, the whole visual system — is already baked into the prompt in Step 2.

Per the spec: personal context is too idiosyncratic to generalize, so it belongs in the request, not the reusable template.

2

Copy the Prompt & Ask a Model

This is the whole spec, assembled and ready — it updates live as you type above. Copy it, paste it to any capable model, and ask for the page. If the model can read files, point it at textbook-chrome-blocks.html and textbook-example-blocks.html so it copies the exact CSS instead of rebuilding it. The prompt already asks for exactly one thing back: a single complete HTML file — no explanation, no code fences.
