Turn the Spec Into a Prompt, Turn the Prompt Into a Page

✦

There's no database behind this site, no "save" button that writes to a server — Ologypedia is a static block library, on purpose. So "adding a page" here doesn't mean filling out a form and watching a new row appear. It means assembling one complete, copy-pasteable prompt — content spec plus visual system, topic dropped in — handing it to whatever capable model you're already talking to, and dropping what comes back into this folder. This page just does the assembling.

Registering a New Page

  1. Save the model's output as textbook-<topic>.html inside apps/ologypedia/.
  2. Open it straight from the filesystem (or serve the workspace locally) and check it renders — masthead, figures, footer, all present.
  3. Paste this card into index.html's library grid, in the .lib-grid.full block with the other worked pages:
Index Card Snippet

No rebuild, no deploy step beyond the usual one — apps/ologypedia/ is a static app, so a new file and one pasted link card is the entire "publish."
